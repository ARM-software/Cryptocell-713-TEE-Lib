This function allows you to calculate SHA256 digest of an image with decryption base on AES-CTR, with HW or user key.

Returns
CC_OK on success.
A non-zero value from bsv_error.h on failure. (in this case, hashBuff will be returned clean, while the output data should be cleaned by the user).
Parameters
[in]hwBaseAddressThe base address of the CryptoCell HW registers.
[in]flowThe supported operations are: HASH, AES to HASH, AES and HASH.
[in]keyTypeThe key type to use: Kce, Kceicv, or user key.
[in]pUserKeyA pointer to the user key buffer in case keyType is CC_BSV_USER_KEY.
[in]userKeySizeThe user key size in bytes (128bits) in case keyType is CC_BSV_USER_KEY.
[in]ivBuffA pointer to the IV / counter buffer.
[in]pInputDataA pointer to the input data.
[out]pOutputDataA pointer to the output buffer. (optional – should be null in case of hash only).
[in]dataSizeThe size of the input data in bytes. MUST be multiple of AES block size.
[out]hashBuffA pointer to a word-aligned 32-byte digest output buffer.

This function calculates the SHA-256 digest over contiguous memory in an integrated operation.

Returns
CC_OK on success.
A non-zero value from bsv_error.h on failure.
Parameters
[in]hwBaseAddressThe base address of the CryptoCell HW registers.
[in]pDataInA pointer to the input buffer to be hashed. The buffer must be contiguous. NULL is a valid input only when dataSize is zero.
[in]dataSizeThe size of the data to be hashed, in bytes.
[out]hashBuffA pointer to a word-aligned 32-byte buffer.
