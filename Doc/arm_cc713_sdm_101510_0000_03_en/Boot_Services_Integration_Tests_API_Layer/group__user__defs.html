Detailed Description

Contains user definitions for the boot services integration tests. These definitions are derived either from values taken from the tests data files or values that are burned to the OTP at the beginning of the integration tests.

Macro Definition Documentation

◆ DCU_DEVELOPER_CERT

#define DCU_DEVELOPER_CERT   {0xFFFFFFFF, 0xFFFFFFFF, 0xFFFFFFFF, 0xFFFFFFFF}

Defines the debug-mask and debug-lock values set in secure debug developer certificates. The values must be the same as defined in developer certificate configuration file (e.g. developer_cert.cfg)

◆ DCU_ICV_ROT_ENABLER_CERT

#define DCU_ICV_ROT_ENABLER_CERT   {0x00112299, 0x44556677, 0x8833AABB, 0x66DDEEFF}

Defines the debug-mask and debug-lock values set in secure debug enabler certificates when HBK0 is the root of trust. The values must be the same as defined in enabler certificate configuration file (e.g. enabler_cert.cfg) ICV is the RoT here and the reset override bit (bit 31 in DCU[3]) is cleared (to ensure reset).

◆ DCU_LOCK_DEFAULT_BUFF

#define DCU_LOCK_DEFAULT_BUFF   {0x55551111, 0xAAAAEEEE, 0x2222CCCC, 0xDDDD3333}

Defines the DCU lock default value burned to the OTP in the test. DCU lock default is used to set the DCU_LOCK<x> registers in secure debug flow

◆ DCU_OEM_ROT_ENABLER_CERT

#define DCU_OEM_ROT_ENABLER_CERT   {0x00112299, 0x4455CC77, 0x8833AABB, 0xCCDDEEFF}

Defines the debug-mask and debug-lock values set in secure debug enabler certificates when HBK1/HBK256 is the root of trust. The values must be the same as defined in enabler certificate configuration file (e.g. enabler_cert.cfg) OEM is the RoT and the reset override bit (bit 31 in DCU[3]) is set (to avoid reset).

◆ EKCST_BUFF

#define EKCST_BUFF   {0x11444411, 0x22223333, 0xFEFEFEEE, 0x66778899}

Defines the EKcst key being burned to the OTP in the test. EKcst is used in EKcst decryption process

◆ HBK0_BUFF

#define HBK0_BUFF   {0x502499D1, 0x99546708, 0x56710165, 0x45B56187}

Defines the ICV HBK (128 bit) being burned to the OTP in the test. The value must be the same as the first 16 bytes defined in hbk_icv_oem.bin file. HBK0 is used as a root of trust in secure boot and secure debug flows, it is also being used for SoC-ID calculation

◆ HBK1_BUFF

#define HBK1_BUFF   {0x8527D794, 0x543D5B38, 0x9E54F507, 0x9C744F53}

Defines the OEM HBK (128 bit) being burned to the OTP in the test. The value must be the same as the last 16 bytes defined in hbk_icv_oem.bin file. HBK1 is used as a root of trust in secure boot and secure debug flows, it is also being used for SoC-ID calculation

◆ HBK256_BUFF

#define HBK256_BUFF   {0x8527D794, 0x543D5B38, 0x9E54F507, 0x9C744F53, 0x86250B8F, 0xEE7D4CB3, 0xEB6B6C90, 0x6E9D1B25}

Defines the OEM HBK (256 bit) being burned to the OTP in the test. The value must be the same as defined in hbk_oem_full.bin file. HBK is used as a root of trust in secure boot and secure debug flows, it is also being used for SoC-ID calculation

◆ HUK_BUFF

#define HUK_BUFF   {0x01020408, 0x01010101, 0x01020408, 0x02020202, 0x01020408, 0x04040404, 0x01020408, 0x08080808}

Defines the HUK key being burned to the OTP in the test. The value must be the same as defined in huk_key.txt file. HUK is used for SoC-ID calculation

◆ KCE_BUFF

#define KCE_BUFF   {0x24232221, 0x28272625, 0x2C2B2A29, 0x202F2E2D}

Defines the Kce OEM key being burned to the OTP in the test. The value must be the same as defined in oem_enc_asset.bin file. Kce OEM is used to encrypt images in secure boot flow

◆ KCEICV_BUFF

#define KCEICV_BUFF   {0x222222FF, 0xFFFFFFFF, 0xFF1111FF, 0xFFFFFFFF}

Defines the Kce ICV key being burned to the OTP in the test. The value must be the same as defined in icv_enc_asset.bin file. Kce ICV is used to encrypt images in secure boot flow

◆ KPICV_BUFF

#define KPICV_BUFF   {0x22223333, 0x11444411, 0x66778899, 0xFEFEFEEE}

Defines the Kcp ICV key being burned to the OTP in the test. Kcp ICV is used in EKcst decryption process

◆ SOCID_ICV_OEM_BUFF

#define SOCID_ICV_OEM_BUFF   {0xea4ca2ae, 0xacfd61fa, 0x7c4448be, 0xc46da21c, 0xf5ac15f3, 0xe1236be8, 0x1736cac0, 0x45b86632}

Defines the calculated SoC-ID based on HUK_BUFF, HBK0_BUFF and HBK1_BUFF. The value must be the same as defined in socid_icv_oem.bin. SOCID_ICV_OEM_BUFF is used to compare SoC-ID where in secure debug flow where HBK0 & HBK1 were burned

◆ SOCID_OEM_FULL_BUFF

#define SOCID_OEM_FULL_BUFF   {0x88fa019b, 0x4840e632, 0x79504be9, 0x28d06217, 0xbe41305a, 0x0b34235e, 0xf2207ffe, 0x1d8fc06f }

Defines the calculated SoC-ID based on HUK_BUFF and HBK256_BUFF. The value must be the same as defined in socid_oem_full.bin. SOCID_OEM_FULL_BUFF is used to compare the SoC-ID in secure debug test flow when FULL HBK was burned

◆ USER_DCU_HW_RESET_BITS

#define USER_DCU_HW_RESET_BITS   {0x00000090, 0x00000000, 0x00000000, 0x60000000}

Defines the DCU bits that cause CryptoCell reset. For more details, see HW CIM document.
