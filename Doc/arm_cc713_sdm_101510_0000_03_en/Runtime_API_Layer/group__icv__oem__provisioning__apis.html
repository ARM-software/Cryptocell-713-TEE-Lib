This API securely provisions ICV or OEM assets to devices, using CryptoCell.

This function takes an encrypted and authenticated asset package produced by the ICV or OEM asset-packaging offline utility (using AES-CCM with key derived from Kpicv or Kcp respectively, and the asset identifier), authenticates and decrypts it. The decrypted asset data is returned to the caller.

Note
This function is valid in all LCS. However, an error is returned if the requested key is locked, invalid, or not used.
Returns
CC_UTIL_OK on success.
A non-zero value on failure, as defined in cc_util_error.h.
Parameters
keyTypetype of key to use to decrypt the package
pAssetPkgBuffpointer to the head of the package
assetPackageLenThe length of the entire package. header + asset
pAssetDataA pointer to a buffer that will contain the decrypted asset. When pAssetData == pAssetPkgBuff the decryption will be performed in-place
pAssetDataLenThe length of decrypted asset data
