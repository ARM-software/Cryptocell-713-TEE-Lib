The function decrypts the encapsulated Secret Key passed by the sender according to the ISO/IEC 18033-2 standard [1], sec. 10.2.4 - ECIES-KEM Decryption.

The function does the following

  • Checks that the ephemeral public key of the sender relates to used EC Domain and initializes the Key structure.
  • Calculates the shared secret value SV.
  • Calculates the Secret Keying data using the secret value SV.
Note
The term "sender" indicates an entity, who creates and performs encapsulation of the Secret Key using this function. The term "recipient" indicates an other entity which receives and decrypts the Secret Key.
Note
All used public and private keys must be related to the same EC Domain.
Note
The function may also be used in Key Transport Schemes and partially, in Integrated Encryption Scheme (section 5.8 - ECIES without optional SharedData of ANSI X9.63-2011: Public Key Cryptography for the Financial Services Industry - Key Agreement and Key Transport Using Elliptic Curve Cryptography).
Returns
CC_OK on success.
A non-zero value on failure as defined cc_ecpki_error.h.
Parameters
[in]pRecipPrivKeyA pointer to the recipient's private key.
[in]kdfDerivModeAn enumerator variable, defining which KDF function mode is used KDF1 or KDF2 (as defined in cc_kdf.h).
[in]kdfHashModeAn enumerator variable, defining used HASH function.
[in]isSingleHashModeSpecific ECIES mode definition 0,1 according to ISO/IEC 18033-2 - sec.10.2.
[in]pCipherDataA pointer to the received encrypted cipher data.
[in]cipherDataSizeA size of the cipher data in bytes.
[out]pSecrKeyPointer to the generated Secret Key.
[in]secrKeySizeThe size of the pSecrKey buffer in bytes.
[in]pTempBuffTemporary buffer for internal usage.

The function creates and encrypts (encapsulate) a Secret Key of required size according to the ISO/IEC 18033-2 standard [1], sec. 10.2.3 - ECIES-KEM Encryption. To call this function the macro CC_EciesKemEncrypt should be used.

The function does the following:

  • Generates random ephemeral EC key pair
  • Converts the ephemeral public key to ciphertext
  • Calculates the shared secret value SV
  • Calculates the Secret Keying data using the secret value SV.
Note
The term "sender" indicates an entity, who creates and performs encapsulation of the Secret Key using this function. The term "recipient" indicates another entity which receives and decrypts the Secret Key.
Note
All used public and private keys must be related to the same EC Domain.
Note
The public key of the recipient must be a legal public key - on elliptic curve.
Note
The function may also be used in Key Transport Schemes and partially, in Integrated Encryption Scheme (section 5.8 - ECIES without optional SharedData of ANSI X9.63-2011: Public Key Cryptography for the Financial Services Industry - Key Agreement and Key Transport Using Elliptic Curve Cryptography).
If external ephemeral keys are used, the caller must verify that these keys are valid.
Returns
CC_OK on success.
A non-zero value on failure as defined cc_ecpki_error.h.
Parameters
[in]pRecipUzPublKeyA pointer to the public key of the recipient.
[in]kdfDerivModeAn enumerator variable, defining which KDF function mode is used KDF1 or KDF2 (as defined in cc_kdf.h).
[in]kdfHashModeAn enumerator variable, defining the used HASH function
[in]isSingleHashModeSpecific ECIES mode definition 0,1 according to ISO/IEC 18033-2 - sec.10.2.
[in]pExtEphUzPrivKeyA pointer to external ephemeral private key - used only for testing.
[in]pExtEphUzPublKeyA pointer to external ephemeral public key - used only for testing.
[out]pSecrKeyPointer to the generated Secret Key.
[in]secrKeySizeThe size of the pSecrKey buffer in bytes.
[out]pCipherDataA pointer to the encrypted ciphertext.
[in,out]pCipherDataSizeA pointer to the size of the output cipher data (in) and its actual size in bytes (out).
[in]pTempBuffTemporary buffer for internal usage.
[in]f_rng- Pointer to DRBG function
[in,out]p_rng- Pointer to the random context - the input to f_rng.
[in]pFipsCtxPointer to temporary buffer used in case FIPS or Chinese certification if required (may be NULL for all other cases).
